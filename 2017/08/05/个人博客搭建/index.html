个人博客搭建

发表于 2017-08-05 | 分类于 其他 | | 阅读次数

为什么要写博客？

在写博客之前，我们需要先思考下这个问题，毕竟写博客是要耗费大量精力的，而且是不赚钱的，你能接受吗？
写博客在我看来是一种习惯，可以让我们更好的去总结、分享、交流提升、同时建立个人品牌，最重要的是思考，最困难是坚持。

为什么要搭建个人博客？

有很多的博客平台，如简书、CSDN，做的也都很不错，为什么不用它们呢？
第一点，个人博客+独立域名对于个人品牌的树立更加有帮助。
第二点，可以把博客做成你想要的样子，对博客拥有完全的控制权。

个人博客的成本？

以我的这个博客为例，云服务器+域名的方案，精打细算一年下来是300元，还可以接受吧？
也有更便宜的方案，如果选择静态博客，可以不用购买服务器，直接部署到github pages上，同时可以绑定自己的域名，只要出个几十块钱的域名钱。

1、网站

选择框架：推荐Hexo
1、简单：Hexo是静态博客，环境搭建简单，不用装mysql、apache、php、redis等等。
2、省钱：因为简单，所以服务器配置不用多好，1核1G足够。
3、功能齐全：写博客、皮肤、分类、标签、评论、统计、分享、站内搜索等，涉及博客的相关的功能都有。
具体怎么搭建可以参考这三个链接：Hexo官网、NexT皮肤、搭建过程踩过的坑。

文章管理：hexo-admin插件
编辑文章需要使用markdown语法，具体语法可以参考：markdown语法
文章的编辑、发布可以通过可视化后台来完成，这个后台不是hexo自带，需要安装hexo-admin插件，安装命令贴在了下面，hexo-admin支持markdown、贴图、添加分类、添加标签等常用功能。

github page部署
可以通过hexo插件一键部署到github page上，可以参考github page部署

2、服务器

购买服务器：推荐阿里云服务器
现在基本都是云服务器，很少有人自己买服务器了，云服务器便宜稳定（国外的更便宜，但说不准哪天就被封了，所以老老实实的用国内的）。我买的是阿里云的ECS服务器，1核1G，3年800，如果你是学生，可以9.9/月。
选linux服务器：即使之前没怎么接触过linux，也建议选linux，系统推荐CentOS 6.6~7.2的系统。
配置端口开放：阿里云默认80端口对外不可访问，需要自己手动配置下，在管理控制台->云服务器ECS->实例->管理->本实例安全组可以配置。

网站的部署：nginx
nginx小而快，只要记得下面两个命令即可安装、启动，想深入学习可以看下我的这篇文章nginx。然后把你的静态资源放到/usr/share/nginx/html目录下面，即可通过公网IP进行访问。（ps：资源部署可以使用xshell和xftp工具；html路径可以通过rpm -ql nginx命令查看）

一键部署：sftp
hexo自带的一个插件，可以完成一键部署的操作，通过ssh协议进行传输，配置可以参考hexo sftp配置。

3、域名

服务器搞定后，你就可以通过公网IP访问你的网站了，用IP访问有点low，所以我们还得搞个域名。
购买域名：优先考虑.com .cn的域名，一年小几十块钱，可以通过万网域名购买地址进行购买。

域名备案：买了域名以后你还不能立即使用，需要备案（在中国是必须的，出于安全考虑，怕你干坏事）。备案的过程很简单，可以通过阿里云备案完成，里面有详细的说明，还有视频教程。提交备案信息后，需要等待审核，一般是2周左右时间。

域名绑定：备案完成后，你就可以使用域名了，接下来要做的是和你的服务器IP做绑定，选择管理控制台->云解析DNS->对应域名进行解析操作即可完成绑定，具体如下图。

网站加速：现在你可以通过域名访问你的网站了，但你会发现有点慢，那是因为你的服务器带宽只有1M，为了提升访问速度，推荐使用CDN，通过管理控制台->CDN->添加域名完成对应域名的加速。
注意点一：记得开启性能优化里的页面优化,智能压缩，也就是gzip压缩，能极大的减少网站流量，提升加载速度。
注意点二：添加完域名后，系统会给你一个加速域名CNAME，但你并未真正的加速，因为你的域名还绑定在IP上。你需要到管理控制台->云解析DNS->对应域名，将之前绑定的IP改为加速域名，具体如下图。
